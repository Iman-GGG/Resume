## 2. Dither 噪点背景

**文件：** `components/Dither.tsx`

**你看到的效果：** 页面背景有一层灰色噪点纹理，会缓慢波动，鼠标靠近时噪点会散开。

### 原理拆解

这个效果分**两层处理**，都在 GPU（显卡）上运行：

```
第一层：波浪生成（Fragment Shader）
  输入 → FBM 噪声算法 → 灰度波浪图案

第二层：抖动量化（Postprocessing Effect）
  波浪图案 → Bayer 8x8 矩阵抖动 → 4 级灰度噪点
```

### 第一层：FBM 分形噪声

```glsl
// 核心就是一个多层叠加的噪声函数
float fbm(vec2 p) {
  float value = 0.0;
  float amp = 1.0;
  for (int i = 0; i < 4; i++) {
    value += amp * abs(cnoise(p));  // 每层取噪声的绝对值
    p *= waveFrequency;              // 频率逐层翻倍（细节越来越密）
    amp *= waveAmplitude;            // 振幅逐层衰减（细节越来越淡）
  }
  return value;
}
```

**通俗解释：** 就像把 4 张不同粗细的"云层图"叠在一起——第一层是大块云，第二层是中块，第三层是小块，第四层是细纹。叠起来就是自然界那种不规则但有层次感的纹理。

加上时间变量 `p - time * waveSpeed` 让它动起来，就是你在页面上看到的缓慢波浪。

**鼠标交互：** 计算鼠标位置和每个像素的距离，距离越近，把噪声值往下压（变暗），形成"推开噪点"的效果：

```glsl
float dist = length(uv - mouseNDC);
float effect = 1.0 - smoothstep(0.0, mouseRadius, dist);
f -= 0.5 * effect;  // 靠近鼠标的地方变暗
```

### 第二层：Bayer 抖动

这一步把平滑的灰度变成只有 4 种灰阶的"像素风"噪点。用的是经典的 **Bayer 8x8 矩阵**：

```
Bayer 矩阵（8x8 的前几行）：
 0  48  12  60   3  51  15  63
32  16  44  28  35  19  47  31
 8  56   4  52  11  59   7  55
...

原理：
  每个像素查表得到一个阈值 threshold
  color += threshold * step  → 加一个微小偏移
  然后 floor(color * 3) / 3  → 量化到 4 个等级
```

**通俗解释：** 原来每个像素是 0-255 的平滑灰色。Bayer 矩阵给相邻像素加上不同的微小偏移，再四舍五入到最近的灰阶。结果就是——相邻像素有的亮一点、有的暗一点，宏观上看起来还是原来的灰度，但微观上变成了老式印刷品一样的抖动网点。

### 和 React 的连接

```tsx
// Dither.tsx 在 layout.tsx 中的使用：
<div className='absolute w-full h-dvh ...'>
  <Dither
    waveColor={[0.31, 0.31, 0.31]}  // 波浪颜色（深灰）
    pixelSize={2}                      // 每个"网点"的大小
    colorNum={4}                       // 量化成 4 级灰度
    waveAmplitude={0.3}                // 波浪强度
    waveFrequency={3}                  // 波浪频率
    waveSpeed={0.05}                   // 动画速度
    enableMouseInteraction             // 开启鼠标交互
    mouseRadius={0.3}                  // 鼠标影响半径
  />
</div>
```

## 3. 标题解密动画

**文件：** `components/DecryptedText.tsx`

**你看到的效果：** 日期文字 "Thursday February 5th, 2026 - New York City" 在页面加载时，每个字符像被破解一样从乱码逐个变成正确字母。

### 原理拆解

核心是一个 `setInterval` 定时器 + React 状态管理：

```
动画循环（每 70ms 执行一次）：
  第 1 帧：×Kj3mP!aB Q2w7n9_5L, 2026 - New York City  (全乱码)
  第 2 帧：Th7r×d!y February 5t_, 2026 - New York City  (前 3 个字母解开)
  第 3 帧：ThuRs9a! Febr2ar_ 5t_, 2026 - New York City  (前 6 个字母解开)
  ...
  最后一帧：Thursday February 5th, 2026 - New York City  (全部解开)
```

代码中的关键逻辑：

```tsx
// 1. 决定 "解密方向"：从左到右
const getNextIndex = (revealedSet) => {
  // 'start' 方式就是：已解密的字母数 = 下一个要解密的位置
  return revealedSet.size;  // 第 0 个、第 1 个、第 2 个...
}

// 2. 每帧把「还没解密的位置」替换成随机字符
const shuffleText = (originalText, currentRevealed) => {
  return originalText.split('').map((char, i) => {
    if (char === ' ') return ' ';           // 空格保持不变
    if (currentRevealed.has(i)) return originalText[i]; // 已解密 → 显示正确字母
    return availableChars[random];           // 未解密 → 显示随机乱码
  }).join('');
}

// 3. 定时器驱动整个流程
interval = setInterval(() => {
  setRevealedIndices(prev => {
    const nextIndex = getNextIndex(prev);
    prev.add(nextIndex);        // 解锁下一个字母
    setDisplayText(shuffleText(text, prev));  // 更新显示
    return prev;
  });
}, 70);  // 每 70ms 解锁一个字母
```

**触发时机：** 这个组件用了 `animateOn="view"`，意思是当元素滚动到屏幕可见区域时才触发，不是 hover 才触发。

## 4. 文字淡入模糊效果

**文件：** `components/motion-primitives/text-effect.tsx`

**你看到的效果：** "Prompt to" 和 "Production" 这些大字，逐个字符从模糊到清晰，同时从下方微微上浮。

### 原理拆解

使用了 **Motion (Framer Motion 的继任者)** 的 `staggerChildren`（错开动画）机制：

```tsx
// 预设的动画状态
'fade-in-blur': {
  hidden:  { opacity: 0, y: 20, filter: 'blur(12px)' },  // 初始：透明 + 下移 + 模糊
  visible: { opacity: 1, y: 0,   filter: 'blur(0px)'  },  // 最终：可见 + 归位 + 清晰
}
```

**容器和子元素的配合：**

```
<容器>                              ← variants.container
  staggerChildren: 0.03           ← 每个子元素之间间隔 0.03 秒
  delayChildren: 0.5              ← 整体延迟 0.5 秒才开始

  <子元素-A>  "P"   ← 0.50 秒开始  } 每个字符是一个
  <子元素-B>  "r"   ← 0.53 秒开始  } 独立的 motion.span
  <子元素-C>  "o"   ← 0.56 秒开始  } 各自按 stagger 间隔
  <子元素-D>  "m"   ← 0.59 秒开始  } 依次播放动画
  ...
</容器>
```

**通俗解释：** 就像多米诺骨牌——容器说"开始！"，然后每隔 0.03 秒推倒一块骨牌。每块骨牌的动作是一样的（从模糊变清晰 + 从下方浮上来），但不是同时做，而是一个接一个。

**使用方式：**
```tsx
<TextEffect
  preset="fade-in-blur"        // 预设效果：淡入 + 模糊
  speedSegment={0.3}           // 每段动画的速度
  as="h1"                      // 渲染成 <h1> 标签
  per="line"                   // 按「行」拆分（不拆字符）
>
  Prompt to
</TextEffect>
```

## 5. 逐组入场动画

**文件：** `components/motion-primitives/animated-group.tsx`

**你看到的效果：** 按钮、功能卡片、议程项目等，逐个从下方弹出，带模糊消散和弹簧回弹。

### 原理拆解

和 TextEffect 同样的 stagger 机制，但动画细节不同——用了 **spring 弹簧物理**：

```tsx
// 来自 lib/utils.ts 的 transitionVariants
item: {
  hidden:  { opacity: 0, filter: 'blur(12px)', y: 12 },
  visible: {
    opacity: 1, filter: 'blur(0px)', y: 0,
    transition: {
      type: 'spring',      // 弹簧动画（不是匀速的缓动）
      bounce: 0.3,         // 弹跳 30%（微微超过目标再弹回来）
      duration: 1.5,       // 1.5 秒内完成
    },
  },
}
```

**通俗解释：** 普通 CSS 动画是"从 A 匀速走到 B"。Spring 动画则是"从 A 弹到 B"——就像一根弹簧拉着元素，它会冲过头、弹回来、再冲过头（幅度越来越小），最后停在目标位置。你看到的每个卡片入场时那种"duang~"的微弹感就是这么来的。

**使用方式：**
```tsx
// 按钮组
<AnimatedGroup variants={{ container: { ... }, ...transitionVariants }}>
  <Button>Register Now</Button>    ← 第 1 个入场
  <Button>Contact Host</Button>    ← 第 2 个入场 (延迟 0.05s)
</AnimatedGroup>
```

## 6. 3D 物理工牌

**文件：** `components/ui/lanyard.tsx`

**你看到的效果：** 右侧挂着一个 3D 工牌，它会受重力自然下垂，可以拖拽，松手后像真的一样晃荡回弹。

> 这是整个项目技术上最复杂的组件。它是真实的物理模拟，不是手写的动画。

### 6.1 物理世界的搭建

```tsx
<Physics gravity={[0, -40, 0]}>   // 重力加速度：y 轴向下 40
  <Band />
</Physics>
```

Rapier 是一个用 WASM（WebAssembly）写的 3D 物理引擎，运行在浏览器里。它做的事情和一个游戏引擎一样：每一帧计算所有物体的位置、速度、碰撞。

### 6.2 绳子 = 刚体链条 + 关节约束

绳子不是一根实体，而是 **5 个小球（刚体）用"关节"串成一串**：

```
  [fixed]  ← 固定点（挂在虚空中不动）
    |  RopeJoint（绳关节：限制最大距离）
  [j1]     ← 第 1 段绳子
    |  RopeJoint
  [j2]     ← 第 2 段绳子
    |  RopeJoint
  [j3]     ← 第 3 段绳子
    |  SphericalJoint（球形关节：能绕这个点自由摆动）
  [card]   ← 工牌卡片
```

代码对应：

```tsx
// 创建刚体
<RigidBody ref={fixed} type="fixed" />                     // 固定不动
<RigidBody ref={j1}   type="dynamic"> <BallCollider/> </RigidBody>  // 受物理影响
<RigidBody ref={j2}   type="dynamic"> <BallCollider/> </RigidBody>
<RigidBody ref={j3}   type="dynamic"> <BallCollider/> </RigidBody>
<RigidBody ref={card} type="dynamic"> ... </RigidBody>    // 工牌

// 用关节连接
useRopeJoint(fixed, j1, [[0,0,0], [0,0,0], 1]);    // 前两个参数：连接谁和谁
useRopeJoint(j1, j2, [[0,0,0], [0,0,0], 1]);    // 第三个参数：锚点位置 + 绳长
useRopeJoint(j2, j3, [[0,0,0], [0,0,0], 1]);
useSphericalJoint(j3, card, [[0,0,0], [0,1.45,0]]);  // 卡片连接点偏上 1.45
```

**关键区别：RopeJoint vs SphericalJoint**

- `RopeJoint`：限制两点之间的**最大距离**，像绳子——可以松弛但不能拉长。松手时卡片往下坠，绳子拉紧到极限就停住了。
- `SphericalJoint`：允许绕一个点**自由旋转**，像球窝关节——卡片可以在绳子末端自由摆荡，不会被固定角度。

### 6.3 "掉下来"和"弹回去"

```tsx
const segmentProps = {
  type: 'dynamic',        // dynamic = 受物理引擎控制（重力、碰撞、惯性）
  canSleep: true,         // 静止后进入休眠省性能
  angularDamping: 4,      // 角度阻尼：旋转速度的衰减
  linearDamping: 4,       // 线性阻尼：移动速度的衰减
};
```

**阻尼的作用：**

```
没有阻尼 = 永远摆下去（永动机）
            ╱╲  ╱╲  ╱╲  ╱╲
           ╱  ╲╱  ╲╱  ╲╱  ╲...

有阻尼   = 逐渐衰减停止（现实物理）
           ╱╲  ╱╲  ╱╲  ___
          ╱  ╲╱  ╲╱  ╲╱
                             ← 最终静止
```

这就是你看到的——松手后工牌**摆荡三四下就稳定了**，和真的一模一样。

### 6.4 拖拽的实现

拖拽用了一个巧妙的状态切换：

```tsx
// 卡片刚体的类型动态切换：
type={dragged ? 'kinematicPosition' : 'dynamic'}
```

- `'dynamic'`（默认）→ "你归物理引擎管"——重力、惯性、碰撞全由 Rapier 计算
- `'kinematicPosition'`（拖拽中）→ "你归我代码管"——物理引擎不管了，位置由代码直接设置

**三个事件实现拖拽：**

```tsx
// 1. 按下 → 记录偏移 + 切换模式
onPointerDown={(e) => {
  drag(new Vector3().copy(e.point).sub(card.current.translation()));
  // drag(鼠标位置 - 卡片位置) = 偏移量，这样拖的时候卡片不会"跳"到鼠标中心
}}

// 2. 移动（在 useFrame 里每帧执行） → 把屏幕坐标反投影成 3D 世界坐标
useFrame((state) => {
  if (dragged) {
    vec.set(state.pointer.x, state.pointer.y, 0.5).unproject(state.camera);
    // unproject: 把屏幕 2D → 3D 世界坐标
    card.current?.setNextKinematicTranslation({ x, y, z });
    // 直接设置卡片下一帧的位置（物理引擎跳过计算）
  }
})

// 3. 松手 → 切回 dynamic 模式，物理引擎重新接管
onPointerUp={() => { drag(false) }}
```

**松手时发生了什么：**

```
1. type 从 'kinematicPosition' 切回 'dynamic'
2. Rapier 检测到：这个物体之前被手动移动了，现在恢复物理
3. 重力开始拉它 ↓
4. RopeJoint 限制它不能掉太远
5. 卡片在绳子末端摆荡
6. angularDamping 和 linearDamping 逐渐消耗能量
7. 几秒后静止 → 完成"弹回去"
```

### 6.5 绳子的视觉渲染

物理关节只决定了"运动"，但绳子**看起来什么样**需要额外渲染：

```tsx
// 每帧：从关节位置构建平滑曲线
curve.points[0].copy(j3.current.translation());  // 卡片端
curve.points[1].copy(j2.current.lerped);          // 插值平滑点
curve.points[2].copy(j1.current.lerped);
curve.points[3].copy(fixed.current.translation()); // 固定端

// CatmullRomCurve3：通过 4 个点的平滑曲线（类似贝塞尔的"通过点"版本）
band.current.geometry.setPoints(curve.getPoints(32));  // 生成 32 个采样点
```

`lerped` 是一个平滑技巧：不是直接用物理位置（会抖动），而是用一个插值变量逐渐逼近物理位置，让绳子看起来柔软顺滑。

**绳子材质：**

```tsx
<meshLineMaterial
  color="white"
  useMap                    // 使用纹理贴图
  map={texture}             // 导入的绳子编织纹理 (lanyard.png)
  repeat={[-4, 1]}          // 纹理重复 4 次
  lineWidth={1}             // 线宽
/>
```

## 7. 卡片纹理生成

**文件：** `components/card-template.tsx`

**你看到的效果：** 3D 工牌上有你的名字、城市、日期，而且输入名字后纹理会实时更新。

### 原理

这个组件**不在屏幕上显示任何东西**（`return null`），它只是在后台用 Canvas 2D 画图，生成一张贴到 3D 模型上的纹理图。

```tsx
const captureTexture = async () => {
  const canvas = document.createElement("canvas");  // 创建离屏画布
  canvas.width = 1376;   // 高分辨率
  canvas.height = 1376;

  const ctx = canvas.getContext("2d");

  // 步骤 1：画底图（从 public/card-base-dark.png 加载）
  ctx.drawImage(baseImage, 0, 0, 1376, 1376);

  // 步骤 2：画名字
  ctx.fillStyle = "#ffffff";
  ctx.font = '48px "Geist Mono"';
  ctx.fillText("YOUR NAME", centerX, bottomY);

  // 步骤 3：画城市
  ctx.fillText("NEW YORK", centerX, topY);

  // 步骤 4：画日期
  ctx.fillStyle = "#878787";  // 灰色
  ctx.fillText("05.02.2026", centerX, dateY);

  // 步骤 5：导出为 data URL → 传给 Three.js 当纹理
  const dataUrl = canvas.toDataURL("image/png");
  onTextureReady(dataUrl);
};
```

然后在 `lanyard.tsx` 中，用 Three.js 的 `TextureLoader` 加载这个 data URL：

```tsx
loader.load(cardTextureUrl, (loadedTexture) => {
  loadedTexture.flipY = false;
  setCustomCardTexture(loadedTexture);
});

// 应用到 3D 卡片材质
<meshPhysicalMaterial map={customCardTexture} />
```

**数据流：**
```
用户输入名字 → handleApplyName → captureTexture → Canvas 画图 → data URL
  → TextureLoader 加载 → Three.js 材质纹理 → 3D 工牌更新
```

## 9. URL 加密分享

**文件：** `lib/utils.ts`

**场景：** 你在输入框里填了名字 "iman"，选了深色卡片，点击分享 → 生成一个带加密参数的 URL：

```
https://v0.app/lanyard?u=dmcwZ2RsOmRhcms6aW1hbg
                                ↑ 看起来是乱码
```

### 加密过程

```tsx
// 1. 原始数据
"iman" + "dark"

// 2. 拼接成固定格式
"v0gdl:dark:iman"    // key:variant:username

// 3. 用 TextEncoder 转成 UTF-8 字节（支持 emoji）
[118, 48, 103, 100, 108, 58, 100, 97, 114, 107, 58, 105, 109, 97, 110]

// 4. 转成 base64
"dmcwZ2RsOmRhcms6aW1hbg=="

// 5. 去掉 =，替换 + 和 /（URL 安全）
"dmcwZ2RsOmRhcms6aW1hbg"
```

### 解密过程（访问 /lanyard 页面时）

```tsx
// 1. 从 URL 取参数
const encrypted = searchParams.get("u");

// 2. 逆向还原
"dmcwZ2RsOmRhcms6aW1hbg"
→ 恢复 +/= → base64 解码 → "v0gdl:dark:iman"
→ 验证 key 匹配 → 提取 variant="dark", username="iman"

// 3. 自动填入工牌
<LanyardWithControls defaultName="iman" defaultVariant="dark" />
```

这样做的好处：别人打开你分享的链接，直接看到你的个性化工牌，不需要手动输入。

```tsx
// app/lanyard/page.tsx
export default function LanyardPage() {
  // 从 URL 读取加密参数
  const lanyardData = decryptLanyardData(searchParams.get("u"));
```
